# 常见问题

本页面汇总了使用 Jav 助手时的常见问题和解决方案。

## 安装和配置

### Q: 扩展安装后没有反应？

**A**: 请检查以下几点：

1. 确保在扩展管理页面开启了"开发者模式"
2. 检查扩展是否已启用（开关是否打开）
3. 刷新 JavDB 页面（按 F5 或 Ctrl+R）
4. 检查浏览器控制台是否有错误（按 F12）

### Q: 为什么不能在 Chrome 应用商店安装？

**A**: 根据 Chrome Web Store 政策，包含成人内容的扩展不能上架。你需要：

1. 从 GitHub Releases 下载预编译版本
2. 手动加载到浏览器
3. 参考 [快速开始教程](https://javd-bviewed-docs.vercel.app/guide/quick-start)

### Q: 扩展会自动更新吗？

**A**: 手动安装的扩展不会自动更新，需要：

1. 关注 GitHub Releases 页面
2. 下载新版本
3. 替换旧版本文件夹
4. 在扩展管理页面点击"重新加载"

### Q: 支持哪些浏览器？

**A**: 支持所有基于 Chromium 的浏览器：

- ✅ Google Chrome
- ✅ Microsoft Edge
- ✅ Brave
- ✅ Opera
- ❌ Firefox（不支持）
- ❌ Safari（不支持）

## 功能使用

### Q: 标记按钮没有显示？

**A**: 请检查：

1. 确认在设置中启用了"显示标记按钮"
2. 确保访问的是 JavDB 网站
3. 刷新页面
4. 检查是否与其他扩展冲突

### Q: 快捷键不生效？

**A**: 可能的原因：

1. 焦点不在页面上（点击页面空白处）
2. 光标在输入框中（点击输入框外）
3. 与其他扩展的快捷键冲突
4. 浏览器快捷键优先级更高

### Q: 标记后没有显示？

**A**: 尝试以下方法：

1. 刷新页面
2. 检查是否有其他优先级更高的状态
3. 打开番号库确认数据是否保存
4. 查看浏览器控制台是否有错误

### Q: 如何取消标记？

**A**: 有两种方式：

1. 再次点击相同的标记按钮
2. 在番号库中删除该记录

## 数据管理

### Q: 数据会丢失吗？

**A**: 数据存储在浏览器本地，通常不会丢失，但以下情况可能导致数据丢失：

- 清除浏览器数据
- 卸载扩展
- 浏览器崩溃
- 系统重装

**建议**:
1. 定期导出数据备份
2. 配置 WebDAV 云端同步
3. 保留多个备份版本

### Q: 如何备份数据？

**A**: 三种备份方式：

1. **手动导出**: 设置面板 → 数据管理 → 导出数据
2. **WebDAV 同步**: 配置云端自动同步
3. **浏览器备份**: 备份浏览器配置文件夹

### Q: 导入数据失败？

**A**: 检查以下问题：

1. 文件格式是否正确（必须是 JSON）
2. 文件是否损坏（用文本编辑器打开查看）
3. 文件大小是否过大（建议小于 50MB）
4. 浏览器存储空间是否充足

### Q: 数据占用空间太大？

**A**: 可以通过以下方式减少：

1. 清理图片缓存
2. 清理演员头像缓存
3. 删除长时间未访问的记录
4. 关闭不需要的缓存功能

## WebDAV 同步

### Q: WebDAV 连接测试失败？

**A**: 常见原因和解决方法：

**服务器地址错误**:
- 检查地址是否包含 `https://`
- 检查地址末尾是否需要 `/`
- 确认路径是否正确

**用户名密码错误**:
- 确认用户名（通常是邮箱）
- 确认使用的是应用密码而非登录密码
- 检查是否有特殊字符

**网络问题**:
- 确认能访问服务器网站
- 检查防火墙设置
- 尝试使用代理

### Q: 同步失败怎么办？

**A**: 根据错误代码处理：

- **401 Unauthorized**: 用户名或密码错误
- **404 Not Found**: 服务器地址或路径错误
- **403 Forbidden**: 没有访问权限
- **500 Server Error**: 服务器内部错误，稍后重试

### Q: 多设备数据不一致？

**A**: 解决方法：

1. 确定哪个设备的数据是正确的
2. 在该设备上点击"立即上传"
3. 在其他设备上点击"立即下载"
4. 选择"使用云端数据"覆盖本地

### Q: 推荐使用哪个 WebDAV 服务？

**A**: 推荐顺序：

1. **坚果云** - 国内访问快，稳定可靠
2. **Nextcloud** - 开源自建，完全掌控
3. **TeraCloud** - 日本服务，免费空间大

## 115网盘

### Q: 115推送失败？

**A**: 最常见的原因：

1. **未登录**: 在 https://115.com 登录账号
2. **Cookie 过期**: 刷新登录状态
3. **磁力链接无效**: 检查链接格式
4. **网络问题**: 检查网络连接

### Q: 推送成功但115没有任务？

**A**: 检查以下位置：

1. 115网盘的"传输列表"
2. "已完成"列表
3. 刷新115页面
4. 检查下载目录设置

### Q: 验证码无法通过？

**A**: 尝试以下方法：

1. 手动完成验证
2. 刷新页面后重试
3. 清除浏览器缓存
4. 重新登录115账号

### Q: 自动标记不生效？

**A**: 检查设置：

1. 确认启用了"推送成功后自动标记"
2. 确认推送确实成功了
3. 刷新页面查看状态
4. 查看浏览器控制台错误

## 演员管理

### Q: 演员头像不显示？

**A**: 解决方法：

1. 检查网络连接
2. 清除头像缓存
3. 点击"更新头像"
4. 检查图片链接是否有效

### Q: 同步演员失败？

**A**: 可能的原因：

1. 未登录 JavDB
2. 网络连接问题
3. JavDB 服务器问题
4. 查看控制台错误信息

### Q: 黑名单演员还是显示？

**A**: 注意以下几点：

1. 确认已开启"隐藏黑名单演员"
2. 演员页面不会隐藏（设计如此）
3. 刷新页面
4. 检查演员是否真的在黑名单中

### Q: 过滤不生效？

**A**: 检查：

1. 确认已启用相关设置
2. 刷新页面
3. 检查演员分类是否正确
4. 查看浏览器控制台错误

## 新作品监控

### Q: 检测不到新作品？

**A**: 检查以下设置：

1. 是否有订阅演员
2. 检测间隔设置
3. 手动点击"立即检测"
4. 查看控制台错误信息

### Q: 检测到的作品不准确？

**A**: 调整过滤设置：

1. 检查标签过滤
2. 检查关键词过滤
3. 确认演员订阅状态
4. 调整检测深度

### Q: 通知不显示？

**A**: 检查权限：

1. 浏览器通知权限
2. 系统通知设置
3. 扩展通知设置
4. 勿扰模式是否开启

## 性能问题

### Q: 扩展占用内存太大？

**A**: 优化方法：

1. 关闭不需要的功能
2. 清理缓存数据
3. 减少订阅演员数量
4. 降低检测频率

### Q: 页面加载变慢？

**A**: 可能的原因：

1. 启用了过多增强功能
2. 缓存数据过多
3. 网络请求过多
4. 与其他扩展冲突

**优化建议**:
1. 关闭不需要的页面增强
2. 清理缓存
3. 调整并发请求数
4. 禁用其他扩展测试

### Q: 浏览器卡顿？

**A**: 减少资源占用：

1. 关闭自动预览功能
2. 减少缓存大小
3. 降低图片质量
4. 关闭数据聚合功能

## 隐私和安全

### Q: 扩展会收集我的数据吗？

**A**: 不会。扩展：

- ✅ 所有数据存储在本地
- ✅ 不会上传到任何服务器
- ✅ 不会追踪用户行为
- ✅ 开源代码可审查

### Q: WebDAV 同步安全吗？

**A**: 是安全的：

- 使用 HTTPS 加密传输
- 密码不会明文存储
- 数据只存储在你的云端
- 完全由你控制

### Q: 如何保护隐私？

**A**: 建议措施：

1. 启用截图模糊模式
2. 使用隐私浏览模式
3. 定期清理浏览历史
4. 不在公共电脑使用

## 错误处理

### Q: 出现"脚本错误"提示？

**A**: 尝试以下方法：

1. 刷新页面
2. 重新加载扩展
3. 清除浏览器缓存
4. 更新到最新版本

### Q: 功能突然不工作了？

**A**: 排查步骤：

1. 检查扩展是否启用
2. 刷新页面
3. 重启浏览器
4. 重新安装扩展

### Q: 如何查看错误日志？

**A**: 打开浏览器控制台：

1. 按 F12 打开开发者工具
2. 切换到 Console 标签
3. 查看红色错误信息
4. 截图或复制错误信息反馈
