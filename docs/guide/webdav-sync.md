# 云端同步 (WebDAV)

通过 WebDAV 配置，你可以在多个设备间自动同步数据，再也不用担心数据丢失。

## 什么是 WebDAV

WebDAV 是一种网络文件传输协议，许多云存储服务都支持 WebDAV 访问。通过 WebDAV，扩展可以将数据自动上传到云端，并在其他设备上下载同步。

## 支持的服务

扩展支持所有标准的 WebDAV 服务，包括但不限于：

- **坚果云** (推荐，国内访问快)
- **Nextcloud** (开源自建方案)
- **TeraCloud** (日本服务，稳定)
- **Yandex Disk** (俄罗斯服务)
- **Box.com**
- **ownCloud**
- 其他支持 WebDAV 的服务

## 配置 WebDAV

### 以坚果云为例

#### 1. 获取 WebDAV 信息

1. 登录坚果云网页版
2. 点击右上角头像 → "账户信息"
3. 找到"安全选项" → "第三方应用管理"
4. 点击"添加应用密码"
5. 输入应用名称（如：Jav助手）
6. 记录生成的应用密码

#### 2. 在扩展中配置

1. 打开设置面板
2. 进入"数据同步"选项卡
3. 填写 WebDAV 信息：
   - **服务器地址**: `https://dav.jianguoyun.com/dav/`
   - **用户名**: 你的坚果云账号（邮箱）
   - **密码**: 刚才生成的应用密码
   - **远程路径**: `/JavdBviewed/` (可自定义)

#### 3. 测试连接

1. 点击"测试连接"按钮
2. 等待测试结果
3. 如果显示"连接成功"，说明配置正确

### 其他服务配置

#### Nextcloud

```
服务器地址: https://your-nextcloud.com/remote.php/dav/files/username/
用户名: 你的用户名
密码: 应用密码（在设置中生成）
远程路径: /JavdBviewed/
```

#### TeraCloud

```
服务器地址: https://webdav.teracloud.jp/dav/
用户名: 你的邮箱
密码: 你的密码
远程路径: /JavdBviewed/
```

## 同步设置

### 自动同步

#### 启用自动同步

1. 在"数据同步"选项卡中
2. 开启"启用自动同步"开关
3. 设置同步间隔（建议 30-60 分钟）

#### 同步间隔

- **15 分钟**: 频繁使用，需要实时同步
- **30 分钟**: 推荐设置，平衡同步及时性和性能
- **60 分钟**: 偶尔使用，减少网络请求
- **自定义**: 根据需要设置

### 手动同步

#### 立即上传

1. 点击"立即上传"按钮
2. 扩展会将本地数据上传到云端
3. 显示上传进度和结果

#### 立即下载

1. 点击"立即下载"按钮
2. 扩展会从云端下载最新数据
3. 如果有冲突，会提示选择处理方式

### 同步范围

可以选择要同步的数据类型：

#### 核心数据（必选）

- 观看记录（番号库）
- 用户资料

#### 可选数据

- **演员数据**: 演员库、演员订阅
- **新作品数据**: 新作品订阅和记录
- **系统配置**: 扩展设置、域名配置、搜索引擎
- **日志数据**: 操作日志（不推荐，会增加同步数据量）

## 冲突处理

### 什么是冲突

当两个设备都修改了数据，且还没来得及同步时，就会产生冲突。

### 冲突解决策略

#### 自动合并（推荐）

- 扩展会智能合并两边的数据
- 保留所有标记记录
- 使用最新的状态

#### 使用本地数据

- 保留本地数据，覆盖云端数据
- 适用于确定本地数据正确的情况

#### 使用云端数据

- 使用云端数据，覆盖本地数据
- 适用于本地数据有问题的情况

### 手动处理冲突

如果自动合并失败，会提示手动处理：

1. 查看冲突详情
2. 选择保留哪一方的数据
3. 或者手动合并后再同步

## 版本管理

### 自动备份

WebDAV 同步会自动保留历史版本：

- 每次上传都会创建新版本
- 默认保留最近 10 个版本
- 可以从历史版本恢复数据

### 查看历史版本

1. 在"数据同步"选项卡中
2. 点击"云端文件管理"
3. 查看所有备份文件列表
4. 显示文件大小和修改时间

### 恢复历史版本

1. 在云端文件列表中
2. 点击要恢复的版本
3. 确认恢复操作
4. 数据会被恢复到该版本

### 删除旧版本

1. 在云端文件列表中
2. 选择要删除的版本
3. 点击"删除"按钮
4. 确认删除

## 备份管理

### 备份保留设置

#### 保留天数

设置备份文件的保留时间：

- **7 天**: 短期备份，节省空间
- **30 天**: 推荐设置，平衡空间和安全
- **90 天**: 长期备份，更安全
- **永久**: 不自动删除旧备份

#### 预警阈值

设置备份过期预警：

- 当距离上次备份超过设定天数时
- 扩展会显示提醒
- 建议设置为 3-7 天

### 备份文件命名

备份文件按以下格式命名：

```
javdb-backup-YYYYMMDD-HHMMSS.json
```

例如：`javdb-backup-20240305-143022.json`

## 多设备同步

### 设置多个设备

在每个设备上：

1. 安装扩展
2. 配置相同的 WebDAV 信息
3. 启用自动同步

### 首次同步

#### 主设备（有数据）

1. 配置 WebDAV
2. 点击"立即上传"
3. 等待上传完成

#### 从设备（无数据）

1. 配置相同的 WebDAV
2. 点击"立即下载"
3. 等待下载完成

### 日常使用

- 每个设备会自动同步
- 无需手动操作
- 数据保持一致

## 故障排查

### 连接测试失败

#### 检查服务器地址

- 确保地址正确，包括 `https://` 前缀
- 地址末尾是否需要 `/`
- 是否包含正确的路径

#### 检查用户名密码

- 用户名是否正确（通常是邮箱）
- 密码是否正确（注意应用密码 vs 登录密码）
- 是否有特殊字符需要转义

#### 检查网络

- 是否能访问服务器网站
- 是否需要代理
- 防火墙是否阻止了连接

### 同步失败

#### 查看错误信息

1. 打开浏览器控制台（F12）
2. 查看 Console 标签页
3. 搜索错误信息

#### 常见错误

**401 Unauthorized**
- 用户名或密码错误
- 应用密码已过期

**404 Not Found**
- 服务器地址错误
- 远程路径不存在

**403 Forbidden**
- 没有访问权限
- 需要在服务器端授权

**500 Server Error**
- 服务器内部错误
- 稍后重试

### 数据不一致

如果发现多个设备数据不一致：

1. 确定哪个设备的数据是正确的
2. 在该设备上点击"立即上传"
3. 在其他设备上点击"立即下载"
4. 选择"使用云端数据"覆盖本地

## 高级功能

### 连接诊断

如果遇到连接问题：

1. 点击"连接诊断"按钮
2. 扩展会进行详细检测：
   - DNS 解析
   - 网络连接
   - 认证测试
   - 权限检查
3. 根据诊断结果修复问题

### 增量同步

扩展使用增量同步技术：

- 只上传变更的数据
- 节省流量和时间
- 提高同步效率

### 压缩传输

数据在传输前会自动压缩：

- 减少传输数据量
- 加快同步速度
- 节省存储空间

## 安全性

### 数据加密

- 使用 HTTPS 加密传输
- 密码不会明文存储
- 支持服务器端加密

### 隐私保护

- 数据只存储在你的云端
- 不会上传到第三方服务器
- 完全由你控制

## 使用建议

### 推荐配置

- **同步间隔**: 30 分钟
- **保留天数**: 30 天
- **预警阈值**: 7 天
- **同步范围**: 核心数据 + 演员数据 + 新作品数据

### 最佳实践

1. **定期检查**: 每周检查一次同步状态
2. **多设备验证**: 在不同设备上验证数据一致性
3. **保留本地备份**: 除了云端同步，还要定期导出本地备份
4. **监控空间**: 注意云端存储空间使用情况
